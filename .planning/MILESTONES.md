# Milestones

## v1.0 milestone (Shipped: 2026-03-26)

**Phases completed:** 5 phases, 14 plans, 39 tasks

**Key accomplishments:**

- macOS app scaffold, developer-strict file taxonomy, and deterministic taxonomy tests for the default editor switcher
- Launch Services association client, requested-vs-effective verifier, smoke probe target, and outcome tests for editor handler changes
- Resident menu bar shell with truthful global-text state aggregation and ranked editor sections
- Batch global-text switching with structured verification results and readback-driven menu refresh
- Inline current markers, compact app-only dropdown behavior, and a real rules-window escape hatch for the resident utility
- A real native settings window now replaces the placeholder rules window and exposes launch-at-login through a testable ServiceManagement boundary
- Recommended menu apps are now persisted, reorderable, and consumed directly by the first-level menu, with unchecked editors moving to `More` instead of being backfilled
- The app now supports follow-system, English, and Simplified Chinese, and menu/settings copy follows the selected language through a shared string catalog
- Developer ID signing, notarization, and pre-install artifact verification are now scripted from the repository instead of being left to manual Organizer flows
- The shipped menu bar flow now explains partial switch failures in localized, scope-specific language and the release checklist extends through an installed app build outside Xcode
- The settings-window milestone work now has a canonical verification report, so v1.0 no longer carries an unverified completed phase
- The milestone now carries an explicit, provable v1.0 release contract instead of an unverified promise of credentialed notarization proof
- The active roadmap, state file, closure verification, and milestone audit now agree on a fully closed v1.0 record
